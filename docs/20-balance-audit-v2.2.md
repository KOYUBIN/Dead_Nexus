## 1. v2.2 수정 사항

`sim-harness/harness_body.js` `batchRun()` 버그:

```js
// 이전 (버그):
const humanSpecific = humanRole === 'ghost' ? ghostClasses[i % 6] : blocs[i % 5];
//                                                          ↑
// i가 짝수일 때만 ghost. i % 6는 짝수 i에서 항상 0/2/4 → CIPHER/BROKER/DRIFTER만 샘플링.
// BLADE(idx 1) / RIGGER(idx 3) / MOLE(idx 5) → 영영 P0 안 됨.

// 수정 (v2.2):
let ghostIdx = 0, blocIdx = 0;
// 별도 카운터로 6종/5종 모두 회전
```

이 버그로 v2.1 audit의 클래스별 승률은 **3종 (CIPHER/BROKER/DRIFTER)만 의미 있고 나머지 통계 누락**.

## 4. 발견된 핵심 이슈 (정정)

### 🔴 P0 — 즉시 패치 필요

| # | 이슈 | 11×11 | 5×5 | 권장 조치 |
|---|---|---|---|---|
| 1 | **RIGGER 완전 비기능** | 0% | 0% | botPickCards에서 RIGGER 카드 점수 검토 + 드론 메커닉 디버그 |
| 2 | **DRIFTER 폭주** | 68.8% | 56.3% | 시작 grid_run LV 1 → 0, mini-raid 발동률 50% → 33% |
| 3 | **MOLE 5×5 0%** | 6.3% | 0% | v1.1.1 mini-raid 50%→33% 너프 일부 롤백 (5×5 한정 50% 복원) |

### 🔴 P1 — 임계 재조정

| # | 이슈 | 측정 | 목표 | 조치 |
|---|---|---|---|---|
| 4 | 11×11 Bloc 우세 | 62.5% | 43% | Bloc 자산 임계 60 → 65 |
| 5 | 5×5 Bloc 우세 | 67.5% | 43% | Bloc 자산 임계 60 → 70 (5×5만) |
| 6 | 게임 길이 5.0R | 4.4~5.0R | 6.8R | 임계 인상으로 자연스럽게 +1.5R 가능 |

### 🟡 P2 — 클래스 다양성 (장기)

- BROKER/CARBON 5~6% — 협상 부족 메커닉이 시뮬에서 드러남
- AXIOM 10% — 주식 변동 메커닉 봇 AI 활용도 낮음

## 5. 다음 패치 (v2.3 후보)

```
[ ] RIGGER 디버그 — 왜 한 판도 못 이기나
[ ] DRIFTER grid_run 시작 LV 너프 (1 → 0)
[ ] MOLE 5×5 mini-raid 50% 복원
[ ] Bloc 자산 임계 +5~10 (5×5/11×11 분리)
[ ] 200판 재측정 → 모든 클래스 5~30% 범위 안에 들어오면 v2.3 확정
```

이 패치 후 RIGGER가 5%+ 받고 DRIFTER가 30~40% 권역에 들면 클래스 균형 OK.
Bloc 자산 임계 인상 후 진영 균형 50:50 ±10% 들어오면 진영 균형 OK.

## 6. 임계값 재조정 (v2.2 baseline 반영)

`balance_test.js`의 THRESHOLDS는 **목표가 아닌 알람 범위**. 현재 실측이 목표에 못 미치므로 임계는 그대로 유지하되, baseline 데이터로만 기록.

다음 패치 적용 후 재측정해서 임계 안에 들어오는지 확인하는 용도.
