## 1. 게임 목표

DEAD NEXUS는 **비대칭 승리 조건**을 가진 게임이다.  
Bloc과 Ghost는 다른 조건으로 승리하며, 서로 다른 자원과 행동 체계를 사용한다.

### 1.1 승리 조건 표

| 역할 | 주 승리 조건 | 부 승리 조건 | 즉시 패배 |
|---|---|---|---|
| **Bloc** | 주식 자산 총액 60 도달 | 타 블록 2곳 완전 인수 | 주가 0 + 모든 구역 피탈 |
| **Ghost** | 렙(Rep) 30 + 블록 레이드 2회 성공 | 비밀 목표 3개 달성 | HP 0 + 부활 불가 + 롱 레스트 전 |

### 1.2 게임 종료 조건

다음 중 하나 발생 시 게임 종료:
1. 어느 한 플레이어가 승리 조건 달성
2. 12라운드 종료 (타임리밋)
3. 공권력 트랙 10 도달 → 계엄 선포 → 특수 엔딩

## 2. 시작 설정

### 2.1 플레이어 역할 선택

게임 시작 전, 각 플레이어는 순서대로 역할을 선택한다.

**선택 순서**: 무작위 결정 (d6 굴림, 가장 높은 사람 먼저)

**역할 선택 규칙**:
- 같은 블록을 두 명이 선택할 수 없음
- Ghost 클래스는 중복 선택 불가
- 최소 1명의 Bloc 플레이어 필요 (없을 시 NPC 블록 자동 활성화)

### 2.2 시작 자원

#### Bloc 플레이어

| 자원 | 시작 보유량 |
|---|---|
| 크레딧 (₵) | 8 |
| 주식 토큰 (자기 블록) | 10 |
| 인플루언스 | 3 |
| 시작 구역 | 3 (HQ 1 + Ring 2 지원 2, 블록별 고정 — `docs/10-map-zones.md` §8) |
| 기본 덱 | 12장 (블록별 구성) |
| 패 크기 | 5장 / 라운드 |

#### Ghost 플레이어

| 자원 | 시작 보유량 |
|---|---|
| 크레딧 (₵) | 5 |
| 렙 (Rep) | 5 |
| 계약 슬롯 | 2 |
| HP | 클래스별 최대치 |
| 시작 위치 | Ring 5 변두리 고정 (MOLE은 HELIX 내부 위장, `docs/10-map-zones.md` §9) |
| 기본 덱 | 10장 (클래스별 구성) |
| 패 크기 | 6장 / 라운드 |

### 2.3 맵 초기화

- **표준 11×11 격자**: 121개 구역 타일 배치 (랜덤 또는 프리셋, `docs/10-map-zones.md` §3)
- **튜토리얼 5×5 격자**: 25구역 (입문·Session 00 용, `docs/10-map-zones.md` §15)
- 각 구역에 **이벤트 토큰 1개** 뒤집어 놓기
- 중앙 구역은 **NEXUS** 고정 (11×11: F6 / 5×5: C3)
- **속성 풀** 빈 상태로 시작
- **공권력 트랙** 0에서 시작
- **시그널 다이** 첫 라운드 굴림 준비

## 3. 턴 구조 (1라운드 = 7단계)

모든 플레이어는 매 라운드 다음 7단계를 순서대로 진행한다.

### 3.1 STEP 1 — 시장 개장 (Market Open)

**소요 시간**: 약 3~5분

1. **시그널 다이** 1개 굴림 → 메시 상태 결정
   - MESH UP / MESH DOWN / SURGE / BLACKOUT
2. **주가 자동 조정**
   - 각 블록 구역 수 변화를 반영해 주가 ±1
   - 이전 라운드 레이드 피해 반영 (−3)
3. **자유 주식 거래**
   - 원하는 플레이어 누구나 동시 매수·매도
   - 거래 주문은 순서 없이 동시 처리
   - 주문 충돌 시 더 많은 크레딧 제시한 플레이어 우선

### 3.2 STEP 2 — 뉴스 (Corporate News)

**소요 시간**: 약 1~2분

1. 뉴스 카드 덱에서 **1장 공개**
2. 카드 텍스트 낭독
3. 즉시 효과 적용
4. 일부 뉴스 카드는 전체 라운드 지속 효과

> ⚠ AXIOM의 PREDICTION ENGINE 카드가 활성화되어 있으면 다음 라운드 뉴스 카드를 미리 확인 가능

### 3.3 STEP 3 — 계획 (Simultaneous Planning)

**소요 시간**: 약 5~10분

1. 모든 플레이어 **동시에** 행동 카드 선택
   - **Ghost**: 패 6장에서 **2장** 선택, 엎어서 앞에 놓음
   - **Bloc**: 패 5장에서 **1~3장** 선택 (정방향·사이드웨이·조합)
2. 속성 비용 지불 선언
   - 속성 풀에서 필요 속성 가져와 카드 위에 배치
3. 이니셔티브 결정
   - Ghost: 선택한 2장 중 TOP 카드의 이니셔티브
   - Bloc: 플레이할 카드 TL × 10

> 📌 이 단계에서는 카드 내용을 공개하지 않음 (일부 예외 있음)

### 3.4 STEP 4 — 실행 (Execution)

**소요 시간**: 약 15~30분

이니셔티브 낮은 순서대로 카드 공개·처리.

#### 4.1 순서 결정
1. 모든 플레이어 이니셔티브 숫자 공개
2. 낮은 숫자 → 먼저 행동
3. 동점 시: Ghost 우선 → BLADE 클래스 → SPD 높은 쪽

#### 4.2 카드 처리
1. 플레이어 차례가 되면 카드 공개
2. 속성 비용 풀에 반납 또는 소모
3. 효과 적용 (이동·전투·자원 획득 등)
4. 충돌 발생 시 즉시 해결 (§ 4 충돌 해결 참조)

#### 4.3 [LOSS] / 스캔들 / 상처 처리
- Ghost [LOSS] 카드 사용 → 소각 패로 이동
- Ghost HP 50% 이하 도달 → 상처 카드 1장 덱 삽입
- Bloc 주요 피해 → 스캔들 카드 1장 덱 삽입

### 3.5 STEP 5 — 수익 (Income)

**소요 시간**: 약 3~5분

1. **구역 수입** — 장악 중인 모든 구역에서 자원·속성 획득
2. **주식 배당** — 10% 이상 지분 보유 시 주식 1주당 ₵ 0.5
3. **속성 풀 감쇠** — 풀에 남은 속성 절반으로 감소 (반올림 처리)

### 3.6 STEP 6 — 테크 R&D

**소요 시간**: 약 2분

- 테크 업그레이드 진행 중인 플레이어, 대기 카운터 1 감소
- 0 도달 시 테크 레벨 상승 + 해당 TL 카드 선택·덱 추가

### 3.7 STEP 7 — 결산 (End of Round)

**소요 시간**: 약 3분

1. **이사회 투표** — 25% 이상 지분 보유자들이 블록 정책 결정 (있을 시)
2. **공권력 트랙** 확인
   - 5 도달 → 해당 라운드 수입 없음 (이미 수령했다면 다음 라운드 적용)
   - 10 도달 → 계엄 선포, 레거시 챕터 3 자동 해금
3. **레거시 조건 체크** — 봉투 해금 조건 달성 시 개봉
4. **라운드 카운터** 1 증가
5. 다음 라운드 준비 (소모된 카드 정리, 속성 풀 확인)

## 4. 충돌 해결 (Combat Resolution)

충돌은 다음 상황에서 발생한다:
- 두 플레이어가 같은 구역을 점령 시도
- Ghost가 Bloc 구역을 레이드
- 플레이어가 다른 플레이어를 직접 공격
- NPC와의 전투 (갱단·경찰·경비)

### 4.1 충돌 기본 절차

#### STEP A — 이니셔티브 확정
- Ghost: 선택한 카드 TOP 이니셔티브 숫자
- Bloc: 플레이한 카드 TL × 10
- BLADE 클래스 자동 선공 패시브 적용

#### STEP B — 공격·방어 선언
양측이 동시에 선언:
- 공격측: ATK 스탯 + 사용 카드 보너스 + 속성 풀 보너스
- 방어측: DEF 스탯 + 방어 카드 보너스 + 구역 방어력

#### STEP C — 주사위 판정
양측 각각 d6 1개 굴림 + 속성 보너스

| 상황 | 보너스 |
|---|---|
| IRON 속성 2 이상 풀 보유 | +1 주사위 |
| MESH 속성 2 이상 풀 보유 | 적 DEF −1 |
| BIO 속성 2 이상 풀 보유 | 자신 HP +2 회복 |
| ASH 속성 2 이상 풀 보유 | 구역 피해 +1 |

#### STEP D — 피해 계산
- 최종 공격값 − 최종 방어값 = 피해량
- 피해량 ≤ 0: 피해 없음 (빗맞음)
- 피해량 > 0: 방어측 HP 감소

#### STEP E — 결과 처리
- HP 0: 패배 → 보유 자원 30% 약탈, 시작 구역으로 이동
- 구역 점령 성공: 새 소유자로 변경, 주가 조정
- 상처 카드 / 스캔들 카드 발생 조건 체크

### 4.2 Ghost vs Bloc 레이드 특례

- Ghost가 Bloc 구역 공격 → **레이드(Raid)** 로 처리
- 방어자는 해당 구역의 Veil 레벨과 경비 무기 수치로 방어
- 성공 시:
  - Ghost 렙 +3
  - 해당 블록 주가 −3
  - 구역 내 자원 50% 약탈
  - 블록 덱에 스캔들 카드 1장 삽입
- 실패 시:
  - Ghost HP −3
  - 현상수배 +1
  - 공권력 트랙 +1

### 4.3 특수 전투 상황

#### 3자 이상 충돌
같은 구역에 3명 이상 진입 시:
1. 이니셔티브 순서대로 2자 충돌 반복
2. 최종 생존자가 구역 획득
3. 모두 중상 시 해당 라운드 구역 "황폐 상태" — 아무도 점령 불가

#### 메시 전투 (Mesh Combat)
CIPHER가 메시다이브 카드로 진입 시 별도 레이어에서 전투:
- HACK 스탯이 주 판정 수치
- 물리 ATK 사용 불가
- 베일(Veil) 레벨이 핵심 방어 요소
- 실패 시 실제 HP도 −2 (백래시)

## 5. 자원 관리

### 5.1 기본 자원 6종

| 자원 | 기호 | 보관 한도 | 용도 |
|---|---|---|---|
| 크레딧 | ₵ | 금고에 따라 12~20 | 모든 기본 비용 |
| 데이터 | 📡 | 10 | 해킹, 주식 조작, 첩보 |
| 인플루언스 | 🎙️ | 8 | 주주 투표, 로비 |
| 무기 | 🔩 | 6 (초과 시 공권력 +1) | 충돌, 레이드 |
| 부품 | ⚙️ | 10 | 테크 업그레이드, 장비 제작 |
| 렙 | ★ | 무제한 (Ghost 전용) | 승리 조건, 스킬 해금 |

### 5.2 속성 자원 (별도)

속성은 기본 자원과 별개로 **공용 속성 풀**에 보관된다.  
개인 보관 불가 — 풀에 쌓아두고 필요할 때 가져다 쓴다.

자세한 내용은 `docs/06-attribute-system.md` 참조.

## 6. 공권력 트랙 (Heat Track)

0부터 10까지. 도시의 긴장도를 나타낸다.

### 6.1 상승 조건
- 무기 보유 6 초과: +1 (매 라운드)
- 충돌 승리: +1
- 레이드 성공: +1
- 특정 이벤트 카드

### 6.2 하락 조건
- 뇌물 행동 (₵ 3 소비): −1
- 특정 뉴스 카드
- MOLE의 신분 세탁 카드

### 6.3 단계별 효과

| 트랙 | 효과 |
|---|---|
| 0~2 | 평시 — 효과 없음 |
| 3~4 | 경계 — 무기 거래 비용 +1 |
| 5~6 | 단속 — 수입 구역당 ₵ −0.5 |
| 7~8 | 긴장 — Ghost 전원 이동 −1 |
| 9 | 임박 — 공권력 NPC 3체 출현 (무작위 구역) |
| 10 | **계엄** — 레거시 챕터 3 자동 해금, 특수 엔딩 분기 |

## 7. 레거시 시스템

### 7.1 기본 원칙
- 플레이한 선택은 **영구적으로 캠페인에 반영**
- 카드·구역·캐릭터가 스티커·낙서로 변경됨
- 봉투는 **한 번만** 열 수 있음
- 캐릭터 사망은 기록됨 (Ghost 영구 제외 모드 선택 가능)

### 7.2 챕터 구조

| 3 | Martial Night | 공권력 트랙 10 도달 |

## 8. 예외 상황 처리

### 8.1 동률 상황
- 이니셔티브 동점: Ghost 우선 → BLADE → SPD 높은 쪽 → 가위바위보
- 주식 매수 동점: 더 많은 크레딧 제시자 우선
- 승점 동점: 장악 구역 수 → 주식 자산 → 충돌 승리 수

### 8.2 카드 부족 / 소진
- Ghost: 패 2장 미만 → SHORT REST 강제 발동
- Bloc: 덱의 스캔들 카드 비율 50% 초과 → 해당 블록 효율 반감

### 8.3 플레이어 이탈 (캠페인 모드)
- 캐릭터를 NPC 블록으로 전환
- 이탈 플레이어의 자원은 NPC가 관리
- 복귀 시 그대로 복원 또는 새 캐릭터 생성

## 9. 규칙 우선순위

여러 규칙이 충돌할 때 적용 순서:

1. **레거시 봉투** 내용 (최우선)
2. **뉴스 이벤트 카드** 일회성 효과
3. **카드 텍스트** (개별 카드의 특수 효과)
4. **블록/클래스 패시브**
5. **기본 규칙** (이 문서)
