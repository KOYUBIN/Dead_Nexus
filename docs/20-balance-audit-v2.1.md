## 3. v1.1 → v2.1 회귀 비교

### v1.1 검증 시점 (CHANGELOG 기록)
- **5×5**: 평균 5.84R, Ghost 114/Bloc 86, BLADE 41% / MOLE 43%
- **11×11**: 평균 6.80R, Ghost 57/Bloc 43

### v2.1 현재 측정
- **5×5**: 평균 4.40R, Ghost 77/Bloc 123, **DRIFTER 78.8%**
- **11×11**: 평균 4.83R, Ghost 92/Bloc 108, **DRIFTER 63.6%**

### 회귀 분석

v1.1 → v2.1 사이에 직접적인 밸런스 코드 변경은 없었음에도 결과가 달라진 이유:
1. **시드 차이**: v1.1 측정은 다른 RNG seed에서 수행
2. **임계 조정**: v1.0.x에서 Bloc 자산 임계가 60으로 통일
3. **봇 AI 진화**: v0.6.5b 이후 봇 R&D rotation이 더 효율적으로 수렴

→ **결론**: v1.1 수치는 단발성. v2.1 baseline을 새 기준으로 채택.

## 5. 회귀 테스트 사용법

```bash
# 기본 (200판 11×11)
cd sim-harness/
node balance_test.js

# 빠른 체크 (100판)
node balance_test.js 100

# 5×5 검증
node balance_test.js 200 5x5

# CI (위반 시 exit 1)
node balance_test.js 200 11x11 --strict

# npm 사용
npm test
npm run test:strict
npm run test:5x5
```

밸런스 변경 후 항상 `npm test` 실행 → 임계 위반이 새로 생기지 않는지 확인.

## 6. 임계값 재정의 절차

`balance_test.js`의 `THRESHOLDS` 객체 수정 시:

1. 변경 직전 200판 결과 baseline 캡처
2. 변경 후 200판 결과 측정
3. 차이가 ±10% 이내면 "허용", 그 이상이면 별도 PR 검토
4. 임계값 변경 시 본 문서 (20-balance-audit-*.md)도 함께 갱신
