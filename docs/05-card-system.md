## 개요

DEAD NEXUS는 **두 개의 다른 카드 시스템**을 역할별로 사용한다.
두 시스템은 공용 속성 풀을 공유하므로 완전히 분리되지는 않으며, 교차 플레이가 가능하다.

| 역할 | 참조 시스템 | 핵심 경험 |
|---|---|---|
| **Ghost** | Gloomhaven | 매 턴이 생사의 선택. 카드가 곧 체력. |
| **Bloc** | Mage Knight | 덱을 성장시켜 기업을 키운다. 조합이 곧 전략. |

## 1. Ghost 카드 시스템 (Gloomhaven 방식)

### 1.1 덱 구성

| 유형 | 수량 | 설명 |
|---|---|---|
| 클래스 전용 카드 | 8장 | 고유 능력. 일부 [LOSS] 포함 |
| 공용 카드 | 2장 | 모든 클래스 동일 (MOVE·BASIC ATK) |
| 시작 시 상처 카드 | 0장 | 피해 누적에 따라 추가됨 |
| 레거시 해금 카드 | 최대 4장 | 챕터 클리어 시 덱에 추가 |

**기본 패 크기**: 6장 / 라운드

### 1.2 카드 해부도

```
┌──────────────────────────────────────┐
│  INITIATIVE: 15        [◈M MESH]     │  ← 이니셔티브 + 주 속성
├──────────────────────────────────────┤
│  ▲ TOP ACTION                        │
│  Generates: ◈M                       │  ← 속성 생성
│  MOVE 3 + PORT SCAN                  │
│  대상 구역 베일 강도 확인            │
├──────────────────────────────────────┤
│  ▼ BOTTOM ACTION                     │
│  Cost: ◈M ◈M ◈S                      │  ← 속성 소비
│  [LOSS] DATA SPIKE                   │  ← 소각 표시
│  ATK +5, DEF 무시, MESH 속성 탈취    │
└──────────────────────────────────────┘
```

### 1.3 턴 행동 규칙

매 턴 패에서 **카드 2장 선택** → 다음 중 하나 실행:
1. **TOP + BOTTOM 혼합**: 1번 카드의 TOP 액션 + 2번 카드의 BOTTOM 액션 사용
2. **2장 중 1장만 선택**: 이니셔티브만 보고 행동 순서 결정, 나머지는 Rest 효과

### 1.4 카드 상태 4종

| 상태 | 설명 | 회복 방법 |
|---|---|---|
| **패 (Hand)** | 현재 사용 가능 | — |
| **버린 패 (Discard)** | 사용 후 임시 보관 | Short Rest로 회복 |
| **소각 패 (Lost)** | [LOSS] 카드 사용 후 | Long Rest만 가능 |
| **상처 카드 (Wound)** | 피해 받으면 덱 삽입 | HELIX 시설·Long Rest |

### 1.5 휴식 시스템

**SHORT REST** (턴 소비)
- 버린 패에서 랜덤 1장 영구 파기 (선택 불가)
- 나머지 버린 패 전부 손으로 복귀
- 소각 패는 회복 안 됨

**LONG REST** (챕터·시나리오 종료 시)
- 덱 전체 리셋 (소각 패 포함)
- 상처 카드 제거 (HELIX 거점 방문 필요)
- 레거시 모드: 롱 레스트마다 캐릭터 일지 기록

### 1.6 소진 (Exhaustion)

패에 카드가 **2장 미만**으로 남으면 행동 불가 → **소진 상태**
- 소진된 Ghost는 SHORT REST를 즉시 수행하거나 해당 라운드 건너뜀
- 레거시 캠페인에서 소진은 렙(Rep) −1 패널티

### 1.7 상처 카드 상세

HP가 최대치의 **50% 이하**로 떨어지면 상처 카드 1장이 **덱에 삽입**된다.

```
┌──────────────────────────────────────┐
│  INITIATIVE : —                      │
├──────────────────────────────────────┤
│  ▲ TOP : STUNNED                     │
│  이번 턴 TOP 행동 불가               │
├──────────────────────────────────────┤
│  ▼ BOTTOM : BLEEDING                 │
│  HP −1 추가 손실                     │
└──────────────────────────────────────┘
```

### 1.8 LOSS 카드 예시 (클래스별)

| 클래스 | LOSS 카드 | 효과 |
|---|---|---|
| CIPHER | **ZERO TRACE** | 이번 라운드 모든 추적·감시 무효화. 베일 자동 해제 |
| BLADE | **LAST STAND** | HP 1까지 DEF 무한. 이후 ATK ×3 반격 |
| RIGGER | **OVERLOAD** | 구역 내 모든 함정·드론 즉시 폭발. 아군 포함 |
| BROKER | **BURN THE BRIDGE** | 모든 진행 중 계약 파기. 양측 자원 40% 강제 환수 |
| DRIFTER | **GHOST RUN** | 3칸 이동, 모든 장벽·함정·충돌 완전 무시 |
| MOLE | **IDENTITY COLLAPSE** | 위장신분 전부 소각. 대신 아무 블록 카드 1장 즉시 복사 사용 |

## 2. Bloc 카드 시스템 (Mage Knight 방식)

### 2.1 덱 구성

- **기본 12장 덱**으로 시작
- TL 상승·구역 장악·M&A·레거시로 덱 성장 (최대 20장)
- **패 크기**: 5장 / 라운드 (남은 카드 다음 라운드 이월)

### 2.2 카드 해부도

```
┌──────────────────────────────────────┐
│  VANTA — TL2         [◈M ◈S]         │  ← 블록 + TL + 속성
│  GHOST PROTOCOL                      │
├──────────────────────────────────────┤
│  ▶ MAIN EFFECT     Cost: ◈M ◈S       │  ← 정방향 + 비용
│  이번 라운드 VANTA 행동 전부 비공개  │
├──────────────────────────────────────┤
│  ↻ SIDEWAYS        Cost: 없음        │
│  Generates: ◈M                       │  ← 옆으로 낼 때
├──────────────────────────────────────┤
│  ⊕ COMBO           Cost: ◈M ◈M ◈S    │
│  + ◈S 카드와 함께 낼 때              │  ← 조합 조건
│  확인한 카드 복사 사용                │
└──────────────────────────────────────┘
```

### 2.3 카드 플레이 방식 3종

| 방식 | 비용 | 효과 |
|---|---|---|
| **정방향 (Main)** | 속성 비용 지불 | 카드의 주 효과 발동 |
| **사이드웨이 (Sideways)** | 없음 | 속성 1개 생성 또는 기본 자원 획득 |
| **조합 (Combo)** | 추가 속성 비용 + 조건 카드 필요 | 정방향 + 추가 시너지 효과 |

### 2.4 블록별 사이드웨이 생산

카드를 SIDEWAYS로 낼 때 생산하는 속성:

| 블록 | SIDEWAYS 생산 |
|---|---|
| VANTA | ◈M 또는 ◈S |
| IRONWALL | ◈I 또는 ◈A |
| HELIX | ◈B 또는 ◈V |
| AXIOM | ◈M 또는 ◈V |
| CARBON | ◈V 또는 ◇×2 |

### 2.5 덱 빌딩 성장

| 방법 | 조건 | 추가 카드 |
|---|---|---|
| 테크 레벨 업 | TL 상승 | 해당 TL 전용 카드 2장 중 1장 선택 |
| 구역 장악 | 새 구역 첫 점령 | 해당 구역 타입 보너스 카드 |
| M&A 완료 | 블록 흡수 | 흡수 블록 카드 2장 계승 가능 |
| 레거시 봉투 | 챕터 해금 | Artifact 등급 희귀 카드 |

### 2.6 덱 가지치기 (Pruning)

- 매 챕터 종료 시 덱에서 기본 카드 1장을 영구 제거 가능
- 약한 카드 제거 → 덱이 점점 날카로워짐
- 메이지나이트의 핵심 덱빌딩 철학

### 2.7 스캔들 카드 (Scandal)

Ghost의 상처 카드와 동등한 개념. Bloc 버전.

**발생 조건**:
- 레이드 성공 당할 때
- M&A 방어 실패 시
- 뉴스 이벤트 스캔들 카드 발동 시

```
┌──────────────────────────────────────┐
│  ⚠ SCANDAL                            │
├──────────────────────────────────────┤
│  뽑을 시 즉시 발동:                  │
│  주가 −1, 인플루언스 −1              │
├──────────────────────────────────────┤
│  ↻ SIDEWAYS: 효과 없음 (카드 낭비)   │
└──────────────────────────────────────┘
```

**제거 방법**:
- PR 캠페인 액션 (인플루언스 3 소비)
- SILK(BROKER)의 중개 서비스
- 뉴스 이벤트 "명예 회복" 카드 발동

## 3. 이니셔티브 시스템

### 3.1 이니셔티브 수치

| 역할 | 기준 |
|---|---|
| Ghost | TOP 카드 상단 숫자 |
| Bloc | 플레이 카드 TL × 10 (기본값) |

**낮은 숫자 = 먼저 행동**

### 3.2 이니셔티브 범위

| 범위 | 의미 |
|---|---|
| 1 – 15 | 초고속 (BLADE 특기) |
| 16 – 30 | 빠름 |
| 31 – 50 | 표준 |
| 51 – 70 | 느림 (강력한 카드) |
| 71 + | 최후 행동 (CARBON 중화기) |

### 3.3 동점 처리

1. Ghost 우선 (Bloc보다 먼저)
2. BLADE 클래스 자동 선공
3. SPD 스탯 높은 쪽
4. 가위바위보

## 4. 두 시스템의 교차점

Ghost와 Bloc이 같은 판에서 상호작용하는 지점들.

### 4.1 공용 속성 풀

양쪽 모두 같은 풀을 공유.
- Ghost TOP 액션 → 속성 생성 → 풀에 쌓임
- Bloc SIDEWAYS → 속성 생성 → 풀에 쌓임
- 누구든 비용 지불 시 풀에서 가져다 사용
- 라운드 종료 시 풀 감쇠 (절반)

### 4.2 카드 임대 (Ghost-Bloc 계약)

Bloc이 Ghost를 고용할 때:
- Bloc 카드 1장을 Ghost에게 1라운드 "지원"으로 대여
- Ghost는 해당 카드를 자신의 패에 추가해 사용
- 라운드 종료 후 Bloc에게 반환 (Ghost 덱에 남지 않음)

### 4.3 충돌 시 카드 교차 작동

**Ghost → Bloc 레이드**:
1. Ghost: 2카드 행동 선언 (TOP + BOTTOM)
2. Bloc: 방어 카드 1장 선택 (정방향 또는 사이드웨이)
3. 이니셔티브 비교 (Ghost TOP 숫자 vs Bloc TL × 10)
4. 피해 계산 → 상처/스캔들 카드 위험

**시그널 다이 영향**:
- SURGE 발동 시: Ghost LOSS 카드 효과 2배 + Bloc 조합 효과 2배
- 동시에 상처·스캔들 발생률 2배

## 5. 두 시스템 비교표

| 항목 | Ghost (Gloomhaven) | Bloc (Mage Knight) |
|---|---|---|
| 덱 크기 | 고정 10장 + 상처 카드 | 성장형 12~20장 |
| 패 크기 | 6장 / 라운드 | 5장 (이월 가능) |
| 행동 방식 | 2카드 TOP + BOTTOM | 1장 정방향·사이드웨이·조합 |
| 이니셔티브 | 카드 상단 숫자 | TL × 10 |
| 강력 카드 | [LOSS] 소각 후 사라짐 | Artifact (레거시 해금) |
| 피해 반영 | 상처 카드 덱 삽입 | 스캔들 카드 덱 삽입 |
| 회복 | Short / Long Rest | PR 캠페인 액션 |
| 소진 | 카드 2장 미만 | 스캔들 50% 초과 |
| 성장 | 레거시 카드 추가 | 덱빌딩 교체 |

## 6. 설계 철학

### 왜 두 시스템인가

**Ghost의 삶**은 매 순간이 긴박하다.
- 한 턴의 잘못된 선택이 죽음으로 이어짐 → 글룸헤이븐의 TOP/BOTTOM 긴장감
- 자원이 곧 생명 → LOSS 카드의 "이걸 지금 써야 하나?" 딜레마
- 피해가 누적 → 상처 카드로 덱이 오염되는 압박감

**Bloc의 삶**은 길고 전략적이다.
- 몇 라운드 뒤를 내다보는 장기 계획 → 메이지나이트의 덱빌딩
- 강력한 조합을 설계 → 카드 조합 시너지
- 유연한 자원 운용 → 사이드웨이 플레이로 속성 수급

### 교차 플레이의 의미

Ghost와 Bloc이 **적대적일 수도, 협력적일 수도** 있다는 세계관을 시스템으로 구현.
- 적대: 공용 속성 풀 쟁탈, 구역 레이드
- 협력: 카드 임대, 속성 공급

공용 풀은 "도시 전체가 숨쉬는 방식" — 모두가 그 안에서 움직인다.
