**문서 ID**: `docs/06-attribute-system.md`

## 변경 요약 (v0.3)

**핵심 변경: 공용 풀 → 개인 풀** (v0.5.3 시뮬레이터 플레이테스트 결정)

이유: 공용 풀은 "내가 쌓은 속성을 상대가 써버리는" 혼란이 컸고, 각 플레이어의 전략이 서로 간섭하며 의도치 않은 교차 플레이가 발생했음. 개인 풀로 분리하면서 각자의 자원 관리가 명확해졌고, 속성 보급 카드의 가치가 올라감.

## 개요

DEAD NEXUS의 카드는 Magic: The Gathering의 마나 시스템을 변형한 **6+1 속성 시스템**을 사용한다.  
각 카드는 속성 비용을 지불해 효과를 발동하며, **각 플레이어는 자신만의 개인 속성 풀**을 운영한다. 매 라운드 시작 시 개인 풀은 절반(올림)으로 감쇠한다.

---

## 1. 6+1 속성

| 속성 | 기호 | 테마 | 색상 |
|---|---|---|---|
| **MESH** | ◈M | 해킹·데이터·메시 | 파란색 (#185FA5) |
| **IRON** | ◈I | 물리·전투·중화기 | 회색 (#888780) |
| **VOLT** | ◈V | 전기·에너지·서지 | 노란색 (#854F0B) |
| **SHADE** | ◈S | 은신·정보·기만 | 보라색 (#534AB7) |
| **BIO** | ◈B | 생체·치유·독 | 녹색 (#1D9E75) |
| **ASH** | ◈A | 파괴·연소·초토화 | 빨간색 (#C04828) |
| **GRID** | ◇ | 무색 범용 | 중립 (#888888) |

> GRID(◇)는 MTG의 무색 마나와 동일. 어떤 속성 비용에도 대신 지불 가능.  
> 단, GRID는 속성 조합 보너스를 발동시키지 않는다.

## 2. 속성 생성 방법

### 2.1 Ghost 클래스 패시브

매 라운드 시작 시 자신의 주 속성 1개 자동 생성.

| 클래스 | 주 속성 | 부 속성 |
|---|---|---|
| CIPHER | ◈M | ◈S |
| BLADE | ◈I | ◈A |
| RIGGER | ◈V | ◈I |
| BROKER | ◈S | ◇ |
| DRIFTER | ◈A | ◇ |
| MOLE | ◈S | ◈M |

### 2.2 Bloc 구역 생산

장악 구역 타입에 따라 수익 단계에 생성.

| 구역 | 생산 속성 |
|---|---|
| 데이터허브 | ◈M |
| 무기고 | ◈I |
| 공업지구 | ◈V |
| 슬럼 | ◈S |
| 의료구역 | ◈B |
| 폐허 | ◈A |
| 마켓 | ◇ ×2 |
| 넥서스 (중앙) | 모든 속성 각 1 |

### 2.3 카드 효과로 생성

일부 카드의 TOP 액션 또는 SIDEWAYS 효과가 속성을 생성한다.
생성된 속성은 **공용 속성 풀**에 놓인다.

### 2.4 크레딧으로 전환

₵ 2 → ◇ 1 (누구나 가능, 매 라운드 제한 없음)
₵ 3 → 원하는 유색 속성 1 (블록 플레이어만 가능, 라운드당 1회)

---

## 3. 공용 속성 풀 (Attribute Pool)

중앙 보드에 위치한 공용 풀. 모든 플레이어가 공유.

### 3.1 작동 방식

- Ghost가 TOP 액션으로 속성 생성 → 풀에 쌓임
- Bloc이 사이드웨이로 카드를 내면 블록 기본 속성 풀에 추가
- 누구든 카드 비용 지불 시 풀에서 가져다 씀
- **라운드 종료 시 풀에 남은 속성은 절반으로 감소** (반올림)

### 3.2 감쇠 예시

라운드 종료 시 풀 상태:
- MESH: 7개 → 4개 (7/2 = 3.5 → 4)
- IRON: 3개 → 2개 (3/2 = 1.5 → 2)
- VOLT: 1개 → 1개 (1/2 = 0.5 → 1)
- ASH: 4개 → 2개 (4/2 = 2)

> 감쇠는 다음 라운드를 위한 "잉여 자원 정리" — 무한 축적 방지.

---

## 4. 속성 강도 (Intensity)

같은 속성이 풀에 쌓이면 **강도**가 올라간다.

| 강도 | 보유량 | 효과 |
|---|---|---|
| **Dim** (약) | 1개 | 기본 비용으로 사용 가능 |
| **Active** (보통) | 2~3개 | 해당 속성 카드 효과 +1 |
| **Surging** (강) | 4개 이상 | 해당 속성 [LOSS] 카드 비용 −1, 효과 +2 |

### 4.1 Surging 활용 전략

- Surging 상태는 특정 속성을 집중 생성해야 달성
- 단일 속성에 올인하면 다른 속성 비용 부담
- 4개 이상 쌓이면 강력하지만, 감쇠로 2개로 떨어질 수 있음 → 연속 라운드 유지 어려움

---

## 5. 속성 연쇄 (Chain)

같은 라운드에 **동일 속성 카드를 연속 사용**하면 연쇄 보너스 발동.

| 연쇄 속성 | 보너스 |
|---|---|
| ◈M × 2 | 다음 HACK 판정 자동 성공 |
| ◈I × 2 | 이번 충돌 주사위 +2 |
| ◈V × 2 | 전자장비 무력화 범위 +1구역 |
| ◈S × 2 | 이번 라운드 위치 완전 비공개 |
| ◈B × 2 | HP 회복 효과 2배 |
| ◈A × 2 | 파괴 효과 인접 구역까지 확산 |

### 5.1 3연쇄 보너스

같은 속성 3회 연속 사용 시:
- 속성 풀에서 해당 속성 4개 즉시 소환 (Surging 상태 달성)
- 이번 라운드 해당 속성 카드 모든 효과 2배

---

## 6. 속성 상성 (Affinity)

속성끼리 상성 관계가 있다.  
상성 카드를 같은 턴에 사용하면 **시너지** — 풀에서 대상 속성 1개 제거 (봉쇄 효과).

```
MESH  ──강함──▶  SHADE   (해킹이 은신을 꿰뚫음)
IRON  ──강함──▶  ASH     (물리력이 파괴를 완성)
VOLT  ──강함──▶  BIO     (전기가 생체를 교란)
SHADE ──강함──▶  MESH    (기만이 데이터를 오염)
ASH   ──강함──▶  VOLT    (파괴가 전력을 차단)
BIO   ──강함──▶  IRON    (생체가 기계를 부식)
```

### 6.1 역상성 페널티

역상성 카드를 같은 턴에 사용하면 서로 간섭:
- 자신의 카드 효과 −1 (양쪽 모두)
- 속성 풀 변동 없음

### 6.2 상성 활용

- 상대의 주력 속성을 파악 후 상성 속성으로 풀 고갈
- 예: VANTA가 MESH 풀을 쌓으면 → SHADE 카드로 봉쇄

---

## 7. 시그널 다이 (Signal Die) — 메시 상태

매 라운드 시작 시 **시그널 다이** 1개 굴림.

| 결과 | 메시 상태 | 효과 |
|---|---|---|
| 🔵 MESH UP (1-2) | 메시 활성 | CIPHER·RIGGER·AXIOM 카드 효과 +1 |
| 🔴 MESH DOWN (3-4) | 메시 불안정 | BLADE·DRIFTER·IRONWALL 카드 효과 +1 |
| ⚡ SURGE (5) | 메시 과부하 | 모든 카드 조합 효과 2배. 상처·스캔들 카드 발생 확률 2배 |
| ⚫ BLACKOUT (6) | 메시 차단 | HACK 기반 행동 전부 불가. CIPHER 이니셔티브 +20 |

### 7.1 메시 상태 지속 시간

- 한 라운드 동안만 유효
- 다음 라운드 시작 시 다이 재굴림
- AXIOM TL 3 (예측 알고리즘): 다음 라운드 다이 결과 미리 확인 가능

---

## 8. 카드 비용 표기법

MTG 방식을 단순화한 표기:

| 표기 | 의미 |
|---|---|
| ◈M | MESH 1개 |
| ◈M ◈M | MESH 2개 |
| ◈M ◈S | MESH 1개 + SHADE 1개 |
| ◇ ◇ ◇ | 무색 3개 (유색 속성으로 대체 가능) |
| ◈M + ◇ ◇ | MESH 1개 + 무색 2개 |
| 없음 | 비용 없음 (사이드웨이 등) |

### 8.1 무색(◇) 치환 규칙

- 유색 속성을 ◇ 대신 지불 가능 (예: ◇ 자리에 ◈I 사용)
- 단, 조합 보너스에는 해당 색이 기여하지 않음
- GRID 단독 카드는 모든 속성을 받음

## 11. 속성 전략 가이드

### 11.1 초반 (1~3라운드)
- 주 속성 중심 카드부터 사용
- 구역 장악으로 속성 공급원 확보
- 풀 Surging 도달 시도

### 11.2 중반 (4~7라운드)
- 부 속성과 조합 실험
- 상성 관계 활용해 상대 속성 봉쇄
- 시그널 다이 상태에 따라 유연 대응

### 11.3 후반 (8~12라운드)
- 비싼 [LOSS]·Artifact 카드 결정타
- Surging 유지로 고비용 카드 연사
- M&A 또는 즉시 승리 조건 도전

---

## 12. 설계 원칙

### 왜 속성 시스템인가

- **비용 지불 메커니즘** → 매 카드가 의미 있는 선택이 됨
- **공용 풀** → 협력과 경쟁 동시 발생
- **상성** → 메타 전략 깊이
- **시그널 다이** → 라운드마다 다른 전장 환경

### MTG와의 차이점

| 항목 | MTG | DEAD NEXUS |
|---|---|---|
| 마나 생성 | 자기 카드(토지)로만 | 공용 풀 공유 |
| 마나 색깔 | 5종 | 6종 |
| 무색 마나 | 기본 비용 절반 이상 | 제한적 사용 |
| 상성 | 없음 | 6개 상성 관계 |
| 라운드 제약 | 자기 차례 끝에 리셋 | 감쇠 (절반) |
