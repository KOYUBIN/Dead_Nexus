# Contributing to DEAD NEXUS

DEAD NEXUS는 현재 **Alpha 단계**이며, 프로젝트 메인테이너 중심으로 설계가 진행되고 있습니다. 외부 기여는 제한적으로 받고 있으며, v0.5부터 본격적으로 개방할 예정입니다.

---

## 현재 받고 있는 기여

### ✅ 지금 환영하는 것

- **오타·문법 수정** — 모든 `.md` 문서의 교정
- **규칙 명확성 피드백** — 애매한 규칙 포착 및 이슈 등록
- **플레이테스트 리포트** — 실제 세션 기록
- **밸런스 이슈 제보** — 수치 이상 또는 남용 가능 조합
- **용어 번역 개선** — 한국어 버전 자연스러움 향상

### ❌ 아직 받지 않는 것

- **새 카드·클래스·블록 추가** — 기획 단계 완료 전에는 보류
- **게임 시스템 전면 변경 제안** — v0.5 이전에는 디자이너 재량
- **아트·일러스트** — 공식 아트 방향 확정 후 별도 공모 예정

## 이슈 제기 (GitHub Issues)

### 이슈 템플릿

**버그·규칙 오류 제보**
```markdown
## 발견한 문제
(규칙이 모순되거나 애매한 지점)

## 참조 문서
docs/02-core-rules.md § 4.2

## 기대한 동작
(명확했다면 어떻게 해석해야 할지)

## 재현 상황
(어떤 플레이 상황에서 문제가 발생했는지)
```

**플레이테스트 리포트**
```markdown
## 세션 정보
- 버전: v0.2.1
- 참가 인원: 4명
- 플레이 시간: 3시간 20분
- 진행자: (이름 또는 닉네임)

## 역할 구성
- P1: Bloc (VANTA)
- P2: Bloc (IRONWALL)
- P3: Ghost (CIPHER / STATIC)
- P4: Ghost (BLADE / RUST)

## 주요 이벤트
(인상 깊은 장면 2~3개)

## 밸런스 관찰
(어떤 전략이 과하게 강하거나 약했는지)

## 규칙 문제점
(플레이 중 발견한 애매한 규칙)

## 개선 제안
(밸런스·규칙 조정 아이디어)
```

**밸런스 이슈**
```markdown
## 문제 카드·시스템
(예: HELIX의 CLONE DECOY + FIELD SPLICE 콤보)

## 관찰된 동작
(승률 거의 100%, 다른 전략 무력화 등)

## 재현 조건
(어떤 상황에서 문제가 생기는지)

## 제안 수정
(비용 증가 / 효과 감소 / 조건부 변경 등)
```

## 풀 리퀘스트 (Pull Requests)

### 브랜치 규칙

- `main`에 직접 푸시 금지
- feature 브랜치 생성 후 PR 요청

```bash
git checkout -b fix/typo-rulebook-section-4
# 작업...
git commit -m "docs(rules): fix typo in section 4.2"
git push origin fix/typo-rulebook-section-4
```

### 커밋 메시지 형식

`<type>(<scope>): <subject>`

| Type | 용도 |
|---|---|
| `feat` | 새 규칙·카드·기능 |
| `fix` | 버그·오류 수정 |
| `docs` | 문서만 변경 |
| `refactor` | 구조 변경 (의미 동일) |
| `balance` | 밸런스 조정 |
| `chore` | 빌드·설정·기타 |

**좋은 예시**
```
docs(cards): clarify SHADOW FILE combo timing
fix(rules): remove duplicate initiative tiebreaker rule
balance(attributes): reduce SURGE cost from 5 to 4
```

**나쁜 예시**
```
update
fix bug
수정함
```

### PR 설명 템플릿

```markdown
## 변경 내용
(무엇을 바꿨는가)

## 이유
(왜 필요한가)

## 관련 이슈
Closes #XX

## 영향 범위
- [ ] 규칙서
- [ ] 카드 텍스트
- [ ] 밸런스 수치
- [ ] 문서 구조
- [ ] 용어

## 체크리스트
- [ ] CHANGELOG.md 업데이트
- [ ] 관련 문서 크로스 레퍼런스 확인
- [ ] 용어 사전 (13-glossary.md) 일관성 확인
```

## 문서 작성 가이드

### 마크다운 스타일

- **헤딩 위계** 지킬 것 (H1 → H2 → H3 순차)
- **테이블** 활용 권장 (카드 수치·자원 비교)
- **코드 블록** — 게임 내 메시지·로그 표현 시
- **인용 블록** — 캐릭터 대사·플레이버 텍스트
- **체크리스트** — 개발 진행상황

### 용어 일관성

- 새 용어 도입 시 **반드시** `docs/13-glossary.md` 등록
- 기존 용어 변경 시 전체 문서 일괄 치환 필요
- 영어 고유명사는 대문자 유지 (VANTA, CIPHER, [LOSS] 등)

### 저작권 안전

- Cyberpunk 2077, Shadowrun, Netrunner 등 기존 작품 용어 **금지**
- 실존 기업 이름 사용 금지 (Apple, Google, Microsoft 등)
- 실존 인물 이름 사용 금지
- 참고했으나 변형한 용어는 `13-glossary.md`에 출처 명시

### 버전 관리

- 문서 상단에 `버전`, `최종 수정일` 명시 권장
- 주요 변경 시 `CHANGELOG.md` 업데이트
- 버전 번호: `MAJOR.MINOR.PATCH`
  - 규칙 구조 바뀜 → MINOR
  - 수치 조정 → PATCH

## 플레이테스트 기여

### 참여 방법

1. 현재 버전 프린트 앤 플레이 세트 다운로드 (v0.5 이후 제공)
2. 세션 진행 후 `playtesting/reports/YYYY-MM-DD-session.md` 형식으로 PR 등록
3. 리포트는 `session-00-guide.md` 템플릿 참조

### 리포트 포함 항목

- 세션 메타데이터 (날짜·인원·시간·버전)
- 역할 구성
- 주요 이벤트·결정적 순간
- 규칙 애매점 / 발견한 버그
- 체감 밸런스
- 개선 제안

## 커뮤니티 행동 강령

### 원칙

- **상호 존중** — 피드백은 아이디어에 대해서지, 사람에 대한 것이 아님
- **건설적 토론** — "이건 별로"가 아닌 "이렇게 바꾸면 어떨까?"
- **저작권 존중** — 외부 작품 표절·무단 차용 엄금
- **정직한 기여** — 타인의 작업을 자신의 것인 양 제출하지 않기

### 금지 행위

- 플레이테스트 참가자 개인정보 공개
- 비공개 레거시 봉투 내용 스포일러 (공식 공개 전)
- 상업적 목적의 무단 사용 (라이선스 공개 전)
