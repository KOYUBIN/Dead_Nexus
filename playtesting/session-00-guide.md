# Playtest Session 00 — First Contact Guide

**문서 ID**: `playtesting/session-00-guide.md`  
**버전**: v0.1  
**최종 수정**: 2026-04-21  
**대상**: 최초 플레이테스트 진행자 (GM/호스트)  
**소요 시간**: 총 2.5~3시간 (브리핑 30분 + 플레이 100~120분 + 디브리핑 30분)

---

## 0. 이 문서의 목적

**Session 00은 게임이 아니라 진단이다.**

DEAD NEXUS는 복잡한 시스템이 13개 문서에 걸쳐 있다. 처음부터 풀세팅으로 돌리면 첫 30분 안에 규칙 설명만으로 테이블이 식는다.  
Session 00은 **간소화 모드**로 핵심 루프가 재미있는지, 규칙이 머리에 들어오는지, 비대칭 승리 조건이 실제 작동하는지만 검증한다.

### 검증 목표 (이걸 측정한다)
1. **턴 구조 이해도** — 7단계 턴을 3라운드 후 플레이어가 "어디 단계죠?"를 얼마나 묻는가
2. **카드 선택의 즐거움** — Ghost의 TOP/BOTTOM 선택, Bloc의 카드 조합이 **고민 거리**가 되는가
3. **속성 풀 체감** — 공용 풀이 "훔쳐 쓰기"와 "타이밍 싸움"을 실제로 만드는가
4. **비대칭 긴장감** — Bloc vs Ghost 상호작용이 재미있는가 (한쪽이 지루하면 실패)
5. **중단 지점** — 어느 규칙에서 테이블이 가장 자주 멈추는가

### 검증하지 않는 것 (Session 00에서는 끈다)
- 레거시 시스템 (봉투·스티커·영구 기록)
- 테크 트리 전체 (TL1만 사용, R&D 단계 스킵)
- M&A / 주식 공매도 / 스캔들 카드
- 메시 맵 / 사이버사이코시스
- 뉴스 이벤트 50장 전체 (큐레이션된 15장만 사용)

## 1. 사전 준비 체크리스트

### 1.1 인원
- **최소 3명 / 최적 4명 / 최대 5명**
- 권장 구성: Bloc 2 + Ghost 2 (4인) 또는 Bloc 1 + Ghost 2 (3인)
- 테이블 관찰자 1명 있으면 피드백 품질이 2배로 뛴다. 가능하면 섭외

### 1.2 준비물
| 항목 | 수량 | 비고 |
|---|---|---|
| **5×5 튜토리얼 맵 보드** | 1 | `docs/10-map-zones.md` §15 프리셋 사용 (25구역) |
| Bloc 기본 덱 | 2~3개 | VANTA·IRONWALL·CARBON 추천 (HELIX·AXIOM은 규칙이 까다로움) |
| Ghost 기본 덱 | 2~3개 | CIPHER·BLADE·BROKER 추천 (MOLE·DRIFTER는 Session 01로 미룸) |
| 뉴스 카드 큐레이션 15장 | 1 덱 | 아래 §5 목록 참조 |
| 속성 풀 토큰 6종 × 10개 | 60개 | 임시 토큰(포커칩) 가능 |
| d6 | 4~6개 | 이니셔티브·전투용 |
| 시그널 다이 1개 | 1 | 없으면 일반 d6 + 대응표 사용 |
| 캐릭터 시트 | 플레이어당 1 | HP·자원 기록용 |
| **피드백 양식 (§7)** | 플레이어당 1 | 프린트 또는 디지털 |
| **관찰자 로그 양식 (§8)** | 1 | 관찰자 있을 때만 |
| 타이머 | 1 | 스마트폰 가능 |
| 스낵·음료 | 충분히 | 3시간이다. 진심이다 |

### 1.3 진행자 (당신) 준비
- [ ] `docs/02-core-rules.md` §1~§3 정독
- [ ] 선택된 Ghost/Bloc 카드 모두 숙지 (플레이어 대신 판정 빠르게 내릴 수 있게)
- [ ] 타이머 세팅: 브리핑 30분, 라운드당 15~20분 목표
- [ ] **규칙 모르면 즉흥 판정 후 기록** — 규칙북 뒤적이며 10분 멈추지 말 것. "내가 지금 임의 판정 내렸음"을 피드백 양식에 메모해 두기
- [ ] 마지막 30분은 디브리핑이다. 플레이가 재미있어도 시간 꼭 남길 것

## 3. 3분 세계관 브리핑 스크립트

진행자가 거의 그대로 읽어도 되는 대본.

> "여기는 애시그리드. 2091년, 사이버펑크 도시.  
> 다섯 개 거대 기업(Bloc)이 도시를 굴린다. 그 틈새에 사는 독립 해결사(Ghost)가 있다.  
>  
> **Bloc은 돈으로 이긴다** — 주식 자산 60이면 승리.  
> **Ghost는 명성으로 이긴다** — 렙 30 + 레이드 2회면 승리.  
>  
> 두 팀은 같은 시간, 같은 맵에서 움직인다. 속성은 공용 풀에서 훔쳐 쓴다.  
> 시끄러운 팀이 이기는 게 아니고, 타이밍 잘 잡은 팀이 이긴다.  
>  
> 이번은 첫 테스트다. 규칙 꼬이면 내가 즉흥 판정한다. 재미없으면 말해라. 우리가 여기 있는 이유는 게임을 고치려는 것."

**중요**: 이 시점에서 플레이어가 "레거시가 뭐예요?" "뉴스 카드 설명해주세요" 물어도 **대답하지 말 것**. "지금은 안 써요"로 컷.

## 6. 라운드별 진행자 체크리스트

### Round 1 — "손 잡고 걷기"
- [ ] STEP 1 직접 굴림 보여주기, 시그널 다이 결과 해석 시연
- [ ] STEP 3에서 플레이어마다 앞에 카드 고르는 과정 옆에서 코치
- [ ] STEP 4 첫 전투 발생 시 **반드시 판정 전 과정 소리 내서 설명**
- [ ] 플레이어 질문: 받아주지만 타이머는 계속 돔
- [ ] 라운드 종료 후 **1분만 쉬기** — "지금 어떤 느낌이에요?" 한 바퀴

### Round 2 — "옆에서 보기"
- [ ] 진행자 개입 최소화. 플레이어끼리 규칙 물어보게 유도
- [ ] **관찰 포인트**: 누가 카드를 오래 고민하는가? 누가 즉시 내는가? 패턴 메모
- [ ] 속성 풀 텅 비는 순간 기록 — 빈도·원인

### Round 3 — "플레이어 자치"
- [ ] 이벤트 토큰 공개 시작
- [ ] 처음으로 Ghost 레이드 발생하면 전체 집중 — 이 순간이 "게임 맞나?"의 분기
- [ ] 진행자는 **이 라운드 판정 외엔 입 다물기**

### Round 4 — "승리 조건 체크"
- [ ] 승리 조건 도달 체크 (Bloc 주식 40 / Ghost 렙 20 + 레이드 1회)
- [ ] 미도달 시 Round 5까지 진행
- [ ] 도달 시 **즉시 게임 종료** — 남은 시간 디브리핑에 투입

## 9. 세션 종료 후 진행자 정리 (당일 안에)

피드백이 기억에서 날아가기 전에, 끝나고 **30분 안에** 다음을 기록:

### 9.1 세션 메타데이터
- 일시·장소·플레이어 명단
- 역할 배정
- 최종 라운드 수
- 승자 (있을 시) / 강제 종료 사유 (있을 시)

### 9.2 Top 3 문제점
규칙·UX·밸런스 구분 없이 가장 많이 테이블을 멈추게 한 3가지.  
각 항목마다: (문제) / (가설 원인) / (다음 세션에서 시도할 수정)

### 9.3 Top 3 성공 포인트
플레이어 눈이 가장 빛난 순간 3가지.  
이건 **절대 수정 금지 대상** — 다음 세션에서도 유지할 것.

### 9.4 임의 판정 목록
Session 00에서 진행자가 현장 판정한 것 전부 나열.  
각 판정은 다음 버전 규칙에 반영 또는 정식 판정 필요.

### 9.5 버전 반영
- 수정 사항은 `CHANGELOG.md`의 `[Unreleased]` 섹션에 투입
- 큰 구조적 변경은 별도 브랜치 (`playtest/s00-fixes`)로 분리
- 카드 밸런스 이슈는 해당 카드 파일에 `<!-- S00 피드백: ... -->` 주석으로 임시 기록

- [ ] 턴 구조 이해도 평균 4.0 이상
- [ ] 전반 재미 평점 평균 3.5 이상
- [ ] 임의 판정 5건 이하 (규칙이 충분히 명확해야 함)
- [ ] 다음 세션 참가 의향 평균 4.0 이상

**어느 하나라도 미달이면 Session 00을 동일 세팅으로 한 번 더 진행** (플레이어 바꿔서).  
같은 문제가 반복되면 규칙 수정 후 재테스트. **Session 01 출발 금지**.

Session 01부터 투입할 시스템 (Session 00 통과 후):
- 테크 트리 전체 (TL1~3)
- 이벤트 토큰 전체 40% 비율
- 뉴스 카드 50장 전체
- 퀘스트 덱 투입
- 블록 시작 구역 3라운드 보호 룰

## 11. 자주 터지는 상황 — 진행자 대응 가이드

### 상황 1: 한 Bloc 플레이어가 라운드마다 너무 강해진다
- **원인 가설**: 속성 풀 독점 / 구역 독점 / 카드 조합이 너무 강함
- **현장 대응**: 규칙 유지 (Session 00에서 실시간 밸런싱 금지)
- **기록**: "Round N부터 플레이어 X 지배 시작" — 원인 메모

### 상황 2: Ghost가 계속 죽거나 아무것도 못한다
- **원인 가설**: 속성 비용이 너무 높음 / 초기 HP 부족 / 레이드 성공률 낮음
- **현장 대응**: 동일. 규칙 유지
- **기록**: Ghost 플레이어 총 행동 수·성공 수 집계

### 상황 3: STEP 3 계획 단계가 한없이 길어진다
- **원인 가설**: 카드 텍스트 복잡 / 선택지 과다 / 분석마비
- **현장 대응**: **2분 타이머 강제 도입** (Session 00부터). 시간 초과 시 랜덤 카드 선택
- **기록**: 평균 선택 시간·초과 빈도

### 상황 4: 전투 판정이 매번 느리다
- **원인 가설**: 속성 보너스 적용 혼란 / d6 판정 여러 번
- **현장 대응**: 첫 전투는 느려도 OK. 3번째부터도 느리면 진행자가 **대신 계산**
- **기록**: 전투 1회당 소요 시간 평균

### 상황 5: 플레이어가 조용해지고 휴대폰을 본다
- **원인 가설**: **가장 위험한 신호**. 자기 턴이 없는 구간이 너무 길다는 뜻
- **현장 대응**: 즉시 휴식 선언. 돌아와서 "지금 어떤 기분?" 솔직히 묻기
- **기록**: 시점·상황 상세 기록 — 핵심 설계 문제일 가능성
